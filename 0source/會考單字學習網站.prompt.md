一、專案總覽與技術堆疊
--------------------------------

1. 專案名稱（可用於專案根目錄與標題）：
   - `會考單字學習網站`（可作為專案名稱與瀏覽器標題）。

2. 專案目標：
   - 提供一個專門用來練習「會考高頻單字表」的學習網站。
   - 使用者可以：
     - 以「帳號＋密碼」登入。
     - 選擇要學習的單字範圍（頁碼、出現次數分級、自訂清單等）。
     - 透過「單字總覽、學習卡模式、測驗模式」進行學習。
     - 查看自己的歷史紀錄與常錯單字。
   - 管理者可以：
     - 檢視與管理所有使用者帳號（含密碼，以明碼顯示）。
     - 檢視各使用者的歷史紀錄與常錯單字。
     - 匯入／匯出帳號與歷史紀錄 JSON 檔案，作為「後端 JSON 資料」。

3. 技術堆疊要求：
   - 使用 React + Vite 建立單頁應用程式（SPA）。
   - 使用 React Router，並採用「HashRouter」：
     - 避免在 GitHub Pages / Netlify / Vercel 上產生路由 404 問題。
   - 語言可使用 TypeScript 或 JavaScript，請自行選擇一種並一致使用：
     - 若使用 TypeScript，檔案副檔名為 .tsx / .ts。
     - 若使用 JavaScript，則為 .jsx / .js。
   - 樣式：
     - 可使用原生 CSS、CSS Modules 或 CSS-in-JS，其一即可。
     - 版面風格以「簡潔、易讀、適合國中生與家長使用」為主。
   - 專案必須能：
     - `npm install`（或 `pnpm install` / `yarn install`）後執行 `npm run dev` 啟動開發伺服器。
     - `npm run build` 產出可部署於 GitHub Pages / Netlify / Vercel 的靜態檔案。

4. 部署目標與限制：
   - 專案為純前端／靜態網站，不得依賴任何自建後端或資料庫。
   - 不使用 SSR / Next.js / Nuxt 等需要伺服器支援的框架特性。
   - 所有資料存取一律以：
     - 前端讀取靜態 JSON 檔案（如 words.json）。
     - 前端 LocalStorage。
     - 管理者匯入／匯出 JSON 檔案的方式完成。

三、資料格式與 JSON 規格
--------------------------------

1. 單字資料（words.json）

請設計 words.json 的結構如下（陣列中的每個物件代表一個單字）：

```json
[
  {
    "id": "a-1",             // 唯一識別碼，可自行設計（例如: 單字＋流水號）
    "word": "apple",         // 英文單字
    "posRaw": "[n.] 名詞",   // 原始詞性標註字串
    "meaningZh": "蘋果",     // 中文解釋（可為多義）
    "frequencyGroup": 10,    // 出現次數分級（整數 1–10，10 代表最高頻）
    "page": 3                // 在原 TOP 字表 PDF 中的頁碼
  }
]
```

注意：
- 不需要保存出現年份列表。
- 預期 words.json 內容是從固定格式的 TOP 單字 PDF 轉換而來（透過你將要產生的 Node.js 工具）。
- 前端網站啟動時，可先讀取 `/public/data/words-sample.json` 作為示範資料。實務上使用者可以將新轉換好的 words.json 放到相同位置覆蓋。

2. 使用者與紀錄資料 JSON（邏輯層面）

在前端運作時，所有帳號與歷史紀錄會同時存在：
- 記憶體狀態（React 狀態／Context）
- LocalStorage

管理者在後台可以：
- 從本機匯入 JSON，更新目前前端的資料。
- 將目前前端的完整使用者／紀錄打包成 JSON 檔並下載（匯出）。

請設計一個合理的資料結構，例如（示意，實作時可微調欄位名稱）：

```json
{
  "users": [
    {
      "username": "student01",
      "password": "123456",           // 依需求，明碼
      "isAdmin": false,
      "createdAt": "2025-11-29T10:00:00.000Z",
      "lastLoginAt": "2025-11-29T11:00:00.000Z"
    },
    {
      "username": "admin",
      "password": "admin",            // 管理者帳號
      "isAdmin": true,
      "createdAt": "2025-11-29T09:00:00.000Z",
      "lastLoginAt": "2025-11-29T10:30:00.000Z"
    }
  ],
  "records": [
    {
      "username": "student01",
      "sessionId": "2025-11-29-1",
      "createdAt": "2025-11-29T11:05:00.000Z",
      "selectionCondition": {
        "type": "pageRange",          // pageRange | frequency | alphabet | customList
        "pages": [3, 4],
        "frequencyGroup": null
      },
      "wordCount": 20,
      "quiz": {
        "mode": "enToZh",             // enToZh | zhToEn
        "totalQuestions": 20,
        "correctCount": 16,
        "accuracy": 0.8
      },
      "wrongWords": [
        "apple",
        "banana"
      ]
    }
  ],
  "userSettings": [
    {
      "username": "student01",
      "maxWordsPerSession": 25,
      "defaultSelectionType": "pageRange",
      "defaultQuizMode": "enToZh",
      "defaultTtsMode": "wordOnly",   // wordOnly | wordAndMeaning
      "defaultTtsIntervalSec": 2
    }
  ]
}
```

要求：
- 請在程式中清楚定義這些結構（TypeScript 型別或 JSDoc 註解皆可），並在管理者匯入／匯出時使用同一套結構。
- 前端啟動時，如 LocalStorage 尚無資料，可建立一份預設資料：
  - 建立一個管理者帳號：帳號 `admin`、密碼 `admin`。
  - 其他使用者由管理者在後台新增，或由使用者註冊時自動寫入。

--------------------------------
四、前端網站功能與頁面說明（系統 A：使用者端）
--------------------------------

以下所有頁面在登入後都應透過共用導覽列（Navbar）互相切換。Navbar 必須包含實際可點擊的連結。

### P0：登入／註冊頁（LoginPage）

目的：
- 讓使用者以「名稱（帳號）＋密碼」登入系統。
- 不同角色（一般使用者、管理者）登入後導向不同首頁。

內容與功能：
- 登入表單：
  - 欄位：帳號、密碼。
  - 按鈕：「登入」。
  - 驗證：
    - 帳號或密碼錯誤時，顯示「帳號或密碼錯誤，請再試一次」。
- 註冊區塊：
  - 可用同一頁面切換成「註冊」模式，或顯示在同頁下方。
  - 欄位：帳號、密碼、確認密碼。
  - 按鈕：「建立新帳號」。
  - 驗證：
    - 帳號不得為空，且不可與現有帳號重複。
    - 密碼與確認密碼需一致。
- 登入流程：
  - 若登入帳號為管理者（isAdmin = true）→ 導向 AdminUsersPage（B1）。
  - 若為一般使用者 → 導向 UserDashboard（A1）。
- 其他：
  - 登入成功後，更新該使用者的 lastLoginAt。

### A1：單字範圍選擇／首頁（UserDashboard）

目的：
- 選擇本回合要學習的單字集合，並建立「本回合單字集」。

內容與功能：
- 選擇方式區塊：
  - 方式一：依「頁碼」選擇
    - 輸入「起始頁碼」與「結束頁碼」，允許單一頁或範圍（例如第 3–4 頁）。
  - 方式二：依「出現次數分級」選擇
    - 以下拉選單讓使用者選擇頻率等級（如 10、9、8 等）。
  - 方式三：依「字母排序區間」選擇
    - 例如 A–C、D–F，用於指定字母起訖區間。
  - 方式四：使用「常錯單字自訂清單」
    - 從該使用者歷史紀錄中彙整的常錯單字集合。
- 顯示符合條件的單字總數，並限制實際加入本回合的最大單字數（例如預設 25，可由個人設定 A6 調整）。
- 按鈕：
  - 「建立本次學習清單」：將目前條件下的單字擷取成「本回合單字集」並存入狀態。
  - 「前往單字總覽」：建立清單後跳轉到 A2。
  - 「直接進入學習卡」：建立清單後跳轉到 A3。
  - 「直接進入測驗」：建立清單後跳轉到 A4。
- 若尚未建立任何本回合單字集，A2/A3/A4 應引導使用者先回到 A1。

### A2：單字總覽頁（WordListPage）

目的：
- 以列表方式呈現本回合的所有單字，並提供連續播放與標記。

內容與功能：
- 表格欄位：
  - 單字（英文）
  - 詞性（posRaw）
  - 中文解釋（meaningZh）
  - 出現次數等級（frequencyGroup）
  - 熟悉度標記（例如：未標記／熟悉／不熟）
- 操作：
  - 點擊某列可開啟該單字詳細資訊（彈出小視窗或側邊欄），顯示同樣欄位即可。
  - 可針對每個單字標記「熟悉」或「不熟」。
- 排序／篩選：
  - 依字母排序（A–Z / Z–A）。
  - 依出現次數排序（高→低或低→高）。
  - 篩選只顯示「不熟單字」。
- 連續播放控制：
  - 使用 Web Speech API（或其他前端 TTS 方案）朗讀單字：
    - 播放模式選項：
      - 只朗讀英文單字。
      - 朗讀英文單字後，再朗讀中文解釋。
    - 控制項：
      - 播放 / 暫停
      - 上一個 / 下一個
      - 間隔秒數設定（例如 1–5 秒，可在 A6 設定預設值）。
- 所有 TTS 錯誤需以友善訊息提示（例如瀏覽器不支援時）。

### A3：學習卡模式頁（FlashcardPage）

目的：
- 透過翻牌方式協助記憶。

內容與功能：
- 顯示單一張卡片，提供兩種模式，可切換：
  - 模式一：英→中
    - 正面：英文單字。
    - 背面：詞性＋中文解釋。
  - 模式二：中→英
    - 正面：中文解釋。
    - 背面：英文單字＋詞性。
- 操作：
  - 「翻面」按鈕或點擊卡片本身翻面。
  - 「上一個」、「下一個」按鈕。
  - 「標記為熟悉／不熟」按鈕。
- 進度顯示：
  - 例如「第 3 / 20 個」。
- 使用者離開此頁面時，其熟悉度標記應同步更新到本回合狀態與個人紀錄。

### A4：測驗模式頁（QuizPage）

目的：
- 針對本回合單字進行選擇題測驗。

內容與功能：
- 測驗模式選擇：
  - 模式一：顯示英文，提供 4 個中文解釋選項。
  - 模式二：顯示中文，提供 4 個英文單字選項。
- 題目生成：
  - 題目從本回合單字集中抽出。
  - 錯誤選項由同一回合中的其他單字隨機抽出，避免太離題。
- 作答流程：
  - 使用者點選一個選項後，立即顯示：
    - 答對：以綠色標示，顯示「答對」。
    - 答錯：以紅色標示，顯示「答錯」並標出正確答案。
  - 然後提供「下一題」按鈕，或在短暫延遲後自動跳到下一題。
- 結算結果：
  - 顯示：
    - 總題數
    - 答對題數
    - 正確率（％）
  - 顯示「本回合答錯的單字清單」。
  - 提供按鈕：
    - 「將本回合常錯單字加入自訂清單」：
      - 將錯誤單字加入此使用者的常錯單字集合，以供未來在 A1 選擇「常錯單字」時使用。
- 結束測驗後，需寫入該使用者的歷史紀錄（records）。

### A5：歷史紀錄與常錯單字頁（HistoryPage）

目的：
- 讓使用者回顧自己的測驗歷史與常錯單字。

內容與功能：
- 歷史紀錄列表：
  - 每筆紀錄包括：
    - 測驗日期時間
    - 選字條件概述（頁碼範圍／出現次數等級／字母區間／常錯清單）
    - 本回合單字數
    - 測驗題數
    - 得分與正確率
  - 點選紀錄可展開詳細資訊（例如每個單字是否答對）。
- 常錯單字區：
  - 顯示累積答錯次數較高的單字清單（可排序）。
  - 每個單字可顯示：
    - 英文、詞性、中文解釋、累積錯誤次數。
  - 提供按鈕：
    - 「使用常錯單字建立新學習清單」：
      - 按下後，將目前常錯清單帶入 A1，並自動將選字方式切換為「自訂清單」。

### A6：個人設定頁（SettingsPage）

目的：
- 讓使用者調整個人偏好設定。需注意「同一台裝置可能有多位使用者」的情況。

內容與功能：
- 設定項目：
  - 每回合最大單字數量（例如預設 25，可在範圍 10–50 內調整）。
  - 預設選字方式：
    - 頁碼範圍／出現次數／字母區間／常錯單字。
  - 預設測驗模式：
    - 英→中 或 中→英。
  - 連續播放預設：
    - 播放模式：只念英文／英文＋中文。
    - 單字間間隔秒數。
- 儲存方式：
  - 所有設定必須「以帳號為單位」儲存：
    - 不可使用單一全域設定，避免 A 使用者更改設定影響 B。
  - 建議 LocalStorage key 命名方式：
    - 例如 `wordsite:user:student01:settings`。
- 切換帳號：
  - 當使用者登出時，需將目前設定與暫存狀態寫回 LocalStorage。
  - 新使用者登入時，需讀取該帳號專屬設定。

五、管理者後台功能與頁面說明（系統 B）
--------------------------------

以下頁面僅在登入帳號 isAdmin = true 時可透過導覽列看到。

### B1：使用者帳號管理頁（AdminUsersPage）

目的：
- 管理使用者帳號、密碼與基本統計。

內容與功能：
- 使用者列表表格：
  - 欄位：
    - 帳號
    - 密碼（明碼顯示，依需求）
    - 是否為管理者
    - 建立時間
    - 最近登入時間
    - 累積測驗次數或最近一次測驗日期
- 操作：
  - 「新增使用者」按鈕：
    - 開啟表單：帳號、密碼、是否管理者。
  - 「編輯」按鈕：
    - 可修改密碼、是否管理者（帳號名稱可視情況允許修改）。
  - 「刪除」按鈕：
    - 刪除該使用者與其相關歷史紀錄與設定（需有確認對話框）。
  - 「重設學習紀錄」按鈕：
    - 清空該使用者的歷史紀錄與常錯單字，但保留帳號與密碼。
- 匯入／匯出 JSON：
  - 匯入：

請在 /tools 目錄下實作一個可在開發者本機執行的 Node.js 命令列工具，用來將「固定格式的 TOP 會考高頻單字 PDF 檔」轉換為 words.json。

2. 使用方式（在 README 中也要說明）：

     - `node tools/convert-top-pdf-to-json.mjs input.pdf output.json`
   - 參數說明：
     - 第一個參數：輸入 PDF 檔路徑（input.pdf）。
     - 第二個參數：輸出 JSON 檔路徑（output.json，若省略可預設為 ./public/data/words.json）。

       - 英文單字
       - 詞性與中文解釋（合併為 posRaw ＋ meaningZh）
       - 出現次數分級（可從標題或表格位置推導，具體實作交由你自行設計）。
       - 頁碼
   - 產生符合前述 words.json 規格的陣列。
   - 將結果以 JSON 格式寫入指定輸出檔案。

4. 錯誤處理：
   - 若未提供輸入檔案路徑，需在 Console 顯示使用說明。
   - 若 PDF 結構與預期不同（無法解析出單字），至少要在 Console 顯示警告訊息。

5. 注意事項：
   - 此工具僅用於開發者在本機預先將 PDF 轉換為 JSON，不會在瀏覽器端執行。
